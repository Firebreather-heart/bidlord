System Overview

BidLord is a high-performance auctioning system built with a modern Python stack. It leverages asynchronous task queues for reliability and WebSockets for real-time communication.

The core flow involves users creating Auction Items, which are then automatically converted into live Auctions by a scheduled background task. Bids are processed asynchronously to ensure the API remains responsive under high load, with real-time price updates broadcast to all connected clients.

Core Features

  • Secure JWT Token Authentication
  • User Registration & Profile Management
  • Full CRUD for Auction Items & Images
  • Scheduled, Automated Auction Creation
  • Asynchronous, High-Throughput Bidding
  • Real-time Price Updates via WebSockets

Technology Stack

  • Django & Django REST Framework
  • PostgreSQL for Persistent Data
  • Redis for Caching & Message Brokering
  • Celery for Background Tasks & Scheduling
  • Django Channels for WebSocket Support
  • Docker for Containerization
